# Excel 道具检索工具（中文界面）实现计划

## 目标与约束

* 工具界面信息全部中文展示。

* 表名称与表头翻译：必须接入“外部免费翻译服务/插件”完成翻译，不使用任何“本地内置翻译库/离线词库”直接翻译。

* 输入：上传包含多个 xlsx 的文件夹（Web 端实际采用上传 zip 或多文件上传）。

* 输出：提供处理后的表文件（已重命名、已插入/填充附表元信息、已分类）的下载方式（zip）。

## 我对需求的可执行化解读（必要假设）

1. “上传 excel 表文件夹”：Web 工具难以直接读取用户本地文件夹，采用两种方式之一实现：

   * 方式A（默认）：上传一个 zip（内含多个 xlsx）。

   * 方式B：多选上传多个 xlsx（浏览器支持的情况下）。
2. “所有表的附表”：按工作簿内“所有工作表（sheet）”理解并逐 sheet 处理；如你实际只希望处理特定 sheet（例如第一个 sheet），可在实现时加开关。
3. “名称搜索”：优先在表头为 `name` 的列中做精确匹配；若找不到 `name` 列则退化为全表字符串精确匹配（可切换为包含匹配）。
4. `id/key` 定位：以第1行表头中列名为 `id`、`key`（大小写不敏感）定位列；若缺失则该 sheet 视为不满足定位条件。
5. “返回对应数据列”：按“命中行的整行数据（以表头->值的键值对形式展示）”返回，同时展示命中的文件名、sheet 名、以及第4步的源 `id/key`。

## 技术方案（从零搭建）

### 应用形态

* 本地 Web 工具（中文 UI），建议使用 Python + Streamlit：

  * 优点：实现上传/下载、表格展示与交互成本低；适合“工具型”内部使用。

  * 产物：一个可直接运行的工具项目（带依赖清单与启动说明）。

### Excel 处理

* 使用 `openpyxl` 读写 xlsx（保持格式相对安全）。

* 处理流程按“流水线”编排为 3 类模块：

  1. 解析与扫描（读取文件、遍历 sheet、定位列、查找命中）。
  2. 变换与生成（重命名、插入列、写入 A1\~A3、翻译表头并写入第3行）。
  3. 归类与二次检索（按类型关键字分组、同类表按 id/key 搜索并返回结果）。

### 翻译接入（外部免费翻译插件/服务）

* 设计一个 `Translator` 适配器接口：`translate(text, src, dst) -> translated_text`

* 默认实现：调用外部“免费在线翻译 API”（HTTP）完成翻译（不在本地做翻译）。

  * 首选：MyMemory Translation API（常见可匿名调用，有频率限制）。

  * 备选：LibreTranslate 公共实例（可能不稳定），或允许用户填自建/可用的免费 endpoint。

* 关键点：

  * 缓存：对表名、表头做本地缓存（同词多次出现只调用一次），减少外部请求。

  * 限流/重试：避免触发 API 限制；失败时允许降级为“原文 + 待翻译”并在 UI 提示。

  * 过滤：空字符串、纯数字、已是中文的文本可选择跳过翻译或仍翻译（提供开关）。

## UI 交互设计（中文）

### 输入区

* 上传区域：上传 zip（或多文件 xlsx），并显示已上传文件数量。

* 查询输入：

  * 类型（下拉 + 可编辑词典）：item/equip/hero/monster/npc/pet/skill/chapter/task/drop/shop/language…；支持新增英文->中文映射。

  * 名称（文本）：要搜索的物品名称。

* 翻译设置：

  * 翻译服务选择（默认 MyMemory / 备选 LibreTranslate / 自定义 URL）。

  * 源语言（默认 auto 或 en）、目标语言（zh-CN）。

### 输出区

* 过程状态：正在处理/完成/失败原因（中文提示）。

* 第4步结果：是否找到名称；若找到，展示命中文件+sheet、行号、`id/key`。

* 第6步结果：同类表二次检索的命中列表（可多条），展示：

  * 命中表（文件名/分类/sheet）

  * 源 `id/key`

  * 命中行数据（表头->值）

* 下载：提供“处理后表文件.zip”下载按钮。

## 处理步骤到代码的映射（对应需求 1\~7）

1. 上传 zip/xlsx；填写“类型+名称”；类型词典可增改。
2. 对每个 xlsx：翻译“原表名”得到中文名；输出文件名为 `原表名(中文).xlsx`。
3. 对每个 sheet：

   * 在 A 列左侧插入 1 列（整列插入）。

   * 写入：A1=`##var`，A2=`##type`，A3=`##`，A4\~留空。

   * 读取第1行表头（从 B1 开始），逐列翻译后写入第3行（从 B3 开始）。
4. 在所有表中按“名称”搜索：

   * 优先 name 列精确匹配，否则全表精确匹配。

   * 取命中行的 `id/key` 列值；若全表未命中：返回空并提示“未找到对应名称物品”。
5. 表分类：

   * 基于“原表名”包含类型英文关键字进行归类，命中则标注类型中文；未命中归入“未分类”。
6. 用第1步类型匹配分类：

   * 若存在同类：在该类所有表中按 `id/key` 二次搜索；命中则返回行数据+表名+源 `id/key`；未命中提示“未找到同名称同类型的物品”。

   * 若不存在同类：提示“未找到相同类的物品”。
7. 将第2/3/5处理完成的表打包 zip 提供下载。

## 项目结构（建议）

* `app/`：Streamlit UI

* `core/`：核心逻辑

  * `excel_io.py`：读取/写入/打包

  * `transform.py`：步骤2/3的变换

  * `search.py`：步骤4/6检索

  * `classify.py`：步骤5分类

  * `translate/`：翻译适配器与缓存

* `configs/`：类型词典默认配置（可在 UI 编辑并导入导出）

* `requirements.txt`：依赖

## 验证与验收（我会在实现阶段逐条自测）

* 能成功上传包含多 xlsx 的 zip，并输出可下载 zip。

* 所有 UI 文案为中文。

* 翻译请求确实走外部 HTTP 服务（可在日志/调试模式展示请求次数，不记录任何敏感信息）。

* 插入列与 A1\~A3 写入正确；第3行表头翻译与第1行逐列对应。

* 名称搜索能找到时正确提取 `id/key`；找不到时给出指定提示。

* 分类规则按“原表名包含类型英文关键字”生效；未命中归入未分类。

* 二次检索在同类表里按 `id/key` 返回命中行数据；未命中给出指定提示。

## 需要你确认/可能调整的点（在你审阅计划时一并决定）

* “附表”究竟是“所有 sheet”还是“仅特定 sheet”（默认所有）。

* 名称匹配是“精确”还是“包含/模糊”（默认精确）。

* 翻译服务优先选择：MyMemory（默认）或你指定的“免费翻译插件/服务”。
