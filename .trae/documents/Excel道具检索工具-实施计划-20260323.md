# Excel 道具检索工具（中文界面）实施计划（Plan）

## 目标

实现一个本地可运行的工具，用于：

1. 通过上传 zip（内含多个 xlsx）导入“Excel 表文件夹”。
2. 将所有表文件重命名为 `原表名称(翻译后的中文).xlsx`。
3. 对每个工作簿内所有 sheet 做“附表结构改造”：插入列、写入 A1~A3 标记、把第 1 行表头翻译为中文并写入第 3 行。
4. 以用户输入的“物品名称”在所有表中搜索，提取命中行的 `id`、`key`、以及“修改前第 1 列/修改后第 2 列”的值。
5. 基于“原表名包含类型英文关键字”对表进行分类并标注类型中文名；未命中归入“未分类”。
6. 若用户输入的类型在分类结果中存在同类，则用第 4 步得到的 `id/key/第2列值` 在同类表的对应列二次检索并返回结果。
7. 输出一个可下载的 zip，包含经过第 2/3/5 步处理后的所有 xlsx。

## 强约束（必须满足）

1. 工具界面文字必须为中文。
2. 表名称与表头翻译必须走“外部免费翻译服务/插件”，优先 MyMemory；不得使用任何内置翻译库/离线词库直接翻译。
3. 上传方式必须支持 zip（用户将 excel 文件夹压缩为 zip 上传）。
4. 名称搜索支持用户切换：精准匹配 / 包含匹配。

## 关键假设（用于落地实现）

1. “附表”按“工作簿内所有 sheet”处理；后续可加开关仅处理第一个 sheet。
2. Excel 首行（第 1 行）是原始表头；第 3 行写入中文表头；插入列后原 A 列整体右移到 B 列。
3. `id`、`key` 以表头文本匹配定位（大小写不敏感，去首尾空格）；缺失则该 sheet 仅参与改造与名称搜索（但无法输出该列值）。
4. “第 3 步修改前第 1 列 / 修改后第 2 列”统一按“修改后 B 列”取值（因为修改后在 A 左侧插入了新列，原第 1 列整体右移到 B 列）。
5. 第 4 步的“名称搜索”优先在表头为 `name` 的列进行；若找不到 `name` 列则退化为“全表扫描字符串”。

## 技术路线（从零搭建）

采用 Python + Streamlit 实现本地 Web 工具（中文 UI）。Excel 读写使用 openpyxl。翻译使用 HTTP 调用 MyMemory 免费 API，并做本地缓存与限流重试，避免触发频控。

## 交付物（仓库内新增/变更内容）

1. 可运行的 Streamlit 应用入口（例如 `app/main.py`）。
2. 核心逻辑模块：zip 解析、Excel 变换、搜索、分类、翻译适配器与缓存。
3. 依赖清单（`requirements.txt`）与使用说明（`README.md`）。
4. 最小化的自测脚本或单元测试（验证关键逻辑：插入列、表头翻译写入、搜索与二次检索、zip 打包）。

## 实现步骤（详细到可直接执行）

### 第 0 步：建立项目骨架与依赖

1. 新建目录结构（建议）：
   - `app/`：Streamlit UI
   - `core/`：核心逻辑
     - `zip_io.py`：zip 解包/打包
     - `excel_transform.py`：步骤 2/3
     - `excel_search.py`：步骤 4/6
     - `excel_classify.py`：步骤 5
     - `translate/`
       - `base.py`：Translator 抽象
       - `mymemory.py`：MyMemory 实现
       - `cache.py`：缓存（内存 + 可选落盘）
2. 添加 `requirements.txt`：
   - `streamlit`
   - `openpyxl`
   - `requests`（或标准库 `urllib`；优先 requests，若不允许再调整）
3. 添加 `README.md`：中文说明“如何启动、如何上传 zip、输出结果含义”。

### 第 1 步：翻译模块（外部免费翻译，MyMemory 优先）

1. 定义统一接口 `Translator.translate(text, src_lang, dst_lang) -> str`。
2. 实现 MyMemory 调用：
   - Endpoint：`https://api.mymemory.translated.net/get?q=...&langpair=src|dst`
   - 处理返回字段：优先取 `responseData.translatedText`。
   - 对空文本、纯数字、明显已是中文文本可配置“跳过翻译”策略（默认：跳过以减少请求）。
3. 增加缓存与限流：
   - 缓存 key：`(src_lang, dst_lang, normalized_text)`。
   - 同一词条多次出现只请求一次。
   - 失败重试 1~2 次（指数退避）；仍失败则返回原文并在 UI 汇总提示“翻译失败词条数”。
4. 为 UI 暴露“翻译服务选择”：默认 MyMemory，保留将来扩展其他免费服务的接口（但本次优先只落 MyMemory）。

### 第 2 步：zip 输入输出（上传 zip、输出 zip）

1. 解析上传 zip：
   - 读取 zip bytes 到内存（不写入用户磁盘或仅写临时目录）。
   - 只提取 `.xlsx` 文件；忽略 `~$` 临时文件；忽略子目录路径安全风险（防 Zip Slip）。
2. 输出 zip：
   - 将处理后的每个 xlsx 写到 bytes buffer（内存流）后打包为 zip。
   - 输出给 Streamlit 的下载按钮。

### 第 3 步：Excel 变换（步骤 2 + 步骤 3）

对每个上传的 xlsx：

1. 读取工作簿（openpyxl），遍历所有 sheet。
2. 步骤 2：重命名输出文件名
   - 取“原表名”= 上传文件名去掉扩展名。
   - 调用翻译：`原表名 -> 中文`。
   - 输出文件名：`原表名(中文).xlsx`（对非法文件名字符做替换）。
3. 步骤 3：对每个 sheet 执行结构改造
   - 在第 3 行上方插入 1 行（即插入新第 3 行，原第 3 行及以下整体下移）。
   - 在 A 列左侧插入 1 列（即插入新第 1 列，原 A 列整体右移到 B 列）。
   - 写入标记：
     - A1 = `##var`
     - A2 = `##type`
     - A3 = `##`
     - A4~ 留空（不主动写入，避免破坏原内容）
   - 翻译表头并写入第 3 行：
     - 读取“原始表头行”统一按“插入行列后”的 B1、C1...（因为插入了新列，原表头应在 B1 起）。
     - 对每个非空表头 cell：翻译为中文，并写入同列的第 3 行（B3、C3...）。
     - 若表头为空则对应 B3 也保持为空。
4. 保存工作簿到 bytes（不覆盖原文件）。

### 第 4 步：名称搜索（在所有表中找“物品名称”）

1. 用户输入：
   - 类型：下拉 + 可编辑词典（英文->中文），支持新增。
   - 名称：文本。
   - 匹配模式：精准 / 包含。
2. 搜索策略（对每个 sheet）：
   - 优先定位 `name` 列（按“原始表头行”匹配；大小写不敏感）。
   - 若定位到 name 列：
     - 从数据区开始逐行扫描（从第 2 行或更靠后，按实际空行处理策略），按用户选择“精准/包含”匹配。
   - 若找不到 name 列：退化为全表扫描字符串（逐 cell 转字符串后匹配）。
3. 命中后提取：
   - 行号（Excel 真实行号）。
   - `id` 列值、`key` 列值（若列不存在则为空）。
   - “修改后第 2 列（B列）”值（若为空则为空）。
   - 同时记录命中文件名、sheet 名。
4. 若全量未命中：返回空并在 UI 提示“未找到对应名称物品”。

### 第 5 步：表分类（按原表名包含类型英文关键字）

1. 维护“类型词典”：默认内置 item/equip/hero/... 的英文->中文；UI 允许用户追加/修改。
2. 对每个文件的“原表名”（不含翻译括号部分）：
   - 若 `原表名` 包含某类型英文关键字（大小写不敏感）：归类到该类型中文名。
   - 若匹配多个关键字：按“最长关键字优先”或“用户词典顺序优先”确定一个类别（实现中固定一种策略并在 README 说明）。
   - 未命中归入“未分类”。
3. 在 UI 输出分类统计（每类多少文件）。

### 第 6 步：同类表二次检索（按 id/key/B列值）

1. 用“用户输入类型”在分类结果中找到同类：
   - 若找不到：提示“未找到相同类的物品”，并不做二次检索。
2. 若找到同类：
   - 取第 4 步的源数据：`id`、`key`、`B列值`（可能为空）。
   - 在同类所有表的所有 sheet 中，按列优先级做匹配：
     1) 若源 `id` 非空且目标表存在 `id` 列：先按 `id` 搜索。
     2) 若源 `key` 非空且目标表存在 `key` 列：再按 `key` 搜索。
     3) 若源 `B列值` 非空：按目标表的 B 列搜索。
   - 命中则返回：
     - 命中表（文件名/分类/sheet）
     - 源 `id/key/B列值`（用于溯源展示）
     - 命中行的整行数据（以“表头->值”的形式展示；表头优先用第 3 行中文表头，其次第 1 行原表头）
3. 若同类表中均未命中：提示“未找到同名称同类型的物品”。

### 第 7 步：输出下载（处理后表文件.zip）

1. 将“已重命名 + 已改造 + 已分类统计可追溯”的 xlsx 全量写入 zip。
2. UI 提供下载按钮，文件名示例：`处理后表文件_YYYYMMDD_HHMM.zip`。

## UI 页面结构（全部中文）

1. 标题：Excel 表转换与道具检索工具
2. 上传区：上传 zip（xlsx 文件夹压缩包）
3. 查询区：
   - 类型选择（下拉 + 编辑词典）
   - 名称输入
   - 名称匹配模式：精准 / 包含
4. 执行按钮：开始处理并搜索
5. 结果区：
   - 第 4 步搜索结果（是否命中、命中文件/sheet、行号、id/key/B列值）
   - 分类结果统计（第 5 步）
   - 第 6 步二次检索命中列表（可多条）
   - 提示信息（未找到对应名称物品/未找到相同类的物品/未找到同名称同类型的物品）
6. 下载区：下载处理后 zip

## 验证清单（实现阶段逐条自测）

1. UI 全中文；无英文硬编码提示语。
2. 翻译调用确实走外部 HTTP（MyMemory），并能在“调试信息”里看到请求次数（不记录任何敏感信息）。
3. 输出文件名符合 `原表名(中文).xlsx`。
4. 每个 sheet：A1/A2/A3 写入正确；第 3 行表头中文与第 1 行逐列对应。
5. 名称搜索支持精准/包含切换；找不到时提示“未找到对应名称物品”。
6. 分类规则生效；未命中进入“未分类”。
7. 二次检索按同类表返回命中行数据；未命中提示“未找到同名称同类型的物品”；无同类提示“未找到相同类的物品”。
8. 最终可下载 zip，且能解压打开所有 xlsx。
